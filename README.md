# Nalam — Tamil-first newborn & maternal health screening for ASHA workers

A Progressive Web App that helps community health workers (ASHA / VHN) screen
newborns and mothers during home visits. Tamil-first UI, voice input, works
offline, installable on any Android phone.

- **Frontend**: React + Vite + Tailwind, PWA (offline + installable)
- **Backend**: Flask API (`nalam/backend.py`). Rules make the clinical
  safety decision; **Gemini 2.5 Flash** (via the Google AI Studio API) writes
  the Tamil + English wording. The model is **prompted, not fine-tuned**.
- **Voice**: Browser Web Speech API in Tamil (`ta-IN`) — speech-to-text runs in
  the browser, no server STT.
- **Storage**: localStorage on device — visits don't leave the phone unless the
  worker is online.

## How the AI works (read this before judging the "AI")

This is the honest description of what runs today:

- **The safety decision is rule-based, not a model.** Whether a baby/mother is
  flagged is deterministic logic over the HBNC checklist (any danger sign →
  refer). This is intentional: clinical decisions stay transparent and
  defensible, not delegated to an LLM.
- **The text is generated by Gemini 2.5 Flash via the Google AI Studio API.**
  It is **prompted** (with an output schema + a few-shot example), **not
  fine-tuned and not trained** by us. There is **no on-device model**, **no
  RAG**, and **no multimodal/image inference** in production.
- **Templates are the fallback.** When `GOOGLE_API_KEY` is absent, the API is
  rate-limited, or the model errors, the backend returns visit-day-aware Tamil
  templates so the service stays up and clinically safe.
- **A referral guardrail** rejects any danger-case counselling text that lacks a
  referral instruction and replaces it with a safe fallback (server-side).

### Planned / not built yet
The repo name (`gemma4n`) reflects an early plan to run **Gemma 3n on-device**.
That is **not built** — it lives only as an experimental Kaggle notebook (see
"What's where"). On-device inference, a fine-tuned Tamil model, and photo-based
assessment are **future work**, not current capabilities.

## How it works

```
ASHA worker's phone (PWA)          Render (Flask backend)
 ┌─────────────────────┐            ┌────────────────────────┐
 │  React app          │   HTTPS    │  /assess  /mother-health│
 │  Tamil voice (STT)  ├───────────►│  /audio-text /counselling│
 │  localStorage       │            │   rules → decision       │
 │  Service worker     │            │   Gemini 2.5 Flash → text│
 └─────────────────────┘            └───────────┬────────────┘
       Vercel CDN                                │ Google AI Studio API
                                                 ▼
                                          Gemini 2.5 Flash
```

Voice transcription happens in the browser, so the voice feature works even
when the backend is cold-starting.

## Run locally

### Quick Start (Automated)

**Linux/Mac:**
```bash
bash setup.sh
```

**Windows (PowerShell):**
```powershell
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
.\setup.ps1
```

This automates all setup steps below and verifies production readiness.

### Manual Setup

```bash
# 1. Backend
cd nalam
pip install -r requirements.txt
export GOOGLE_API_KEY=<your Google AI Studio key>   # optional: omit to run templates only
FLASK_ENV=development python backend.py         # http://localhost:5000
# FLASK_ENV=development is required locally so CORS allows http://localhost:5173

# 2. Knowledge Base (first time only)
cd ../knowledge_base
pip install -r requirements.txt
python build-index.py                               # Builds FAISS vector store
cd ../nalam

# 3. Frontend (new terminal)
cd nalam
cp .env.example .env                                 # VITE_API_URL=http://localhost:5000
npm install
npm run dev                                          # http://localhost:5173
```

## Deploy a live demo (~15 min, free)

Live instances:
- App: `https://gemma4n-nalama.vercel.app`
- API: `https://nalam-api-h62b.onrender.com`

### Step 1 — Push to GitHub
Push the repo to GitHub (Vercel and Render both deploy from it).

### Step 2 — Deploy the backend to Render
1. <https://dashboard.render.com> → **New** → **Blueprint**.
2. Connect the repo. Render reads [`render.yaml`](./render.yaml) and proposes a
   service named `nalam-api`.
3. Add the secret **`GOOGLE_API_KEY`** (your Google AI Studio key). Without it
   the API still runs but returns templates only.
4. Click **Apply**. When live, open `/health` — you should see
   `{"status": "Nalam API running", "model": "gemini-2.5-flash"}`
   (or `"rules-only-fallback"` if the key is unset).

> **Free tier note:** Render free instances sleep after 15 min idle (~50 s cold
> start). Gemini's free tier is ~15 req/min — heavy bursts fall back to
> templates. Fine for a demo; upgrade both for a real pilot.

### Step 3 — Deploy the frontend to Vercel
1. <https://vercel.com/new> → import the same repo. It reads
   [`vercel.json`](./vercel.json).
2. Add env var `VITE_API_URL` = your Render URL (no trailing slash).
3. **Deploy.**

### Step 4 — CORS
CORS is locked down **in code** (`backend.py`): the production Vercel origin
is the only allowed origin; localhost is allowed only when
`FLASK_ENV=development`. There is nothing to configure in the Render dashboard
for CORS. (The old `FRONTEND_ORIGIN` env var is no longer used.)

### Step 5 — Install on a phone
On Android, open the Vercel URL in Chrome → menu → **Install app**. It launches
full-screen, works offline, and the Tamil mic uses the phone's microphone.

## Limitations to be honest about

- **Decisions are rules, text is Gemini (prompted, not fine-tuned).** Quality of
  the *wording* depends on the model; the *decision* does not.
- **No emergency-severity grading at `/assess` and `/mother-health`** — they
  return only `is_safe`. Emergency vs PHC is only distinguished at
  `/counselling`. (Tracked by the eval set.)
- **Browser speech recognition** needs Chrome / Edge / Samsung Internet; iOS
  Safari `ta-IN` support is limited.
- **localStorage is per device** — there is no sync between phones.
- Birth weight is **not** part of the danger-sign rule, so a very-low-weight baby
  passing the 4 checks is currently marked safe (a known gap; see the eval set).

## Disclaimer

This is a **decision-support tool**. It does not replace clinical judgment.
All danger-sign paths default to "refer to PHC / call 108" when the system is
uncertain — enforced by rules and a server-side referral guardrail.
