# HBNC Knowledge Base

This directory contains the knowledge base system for the Nalam backend, enabling RAG (Retrieval Augmented Generation) for clinical decision support.

## Directory Structure

- **`raw/`** - Raw markdown source documents covering HBNC protocols, danger signs, nutrition, infections, and follow-up schedules
- **`embeddings/`** - Pre-built FAISS vector store and metadata (generated by `build-index.py`)
- **`build-index.py`** - Script to chunk documents, embed them with Google's Gemini API, and build the FAISS index
- **`retriever.py`** - Module providing RAG query, retrieval, and context formatting for the backend

## Setup

### Prerequisites

```bash
pip install -r requirements.txt
```

### Building the Vector Store

**First time setup:**

1. Set your Google API key:
   ```bash
   export GOOGLE_API_KEY='your-key-from-https://aistudio.google.com/app/apikey'
   ```

2. Build the FAISS index:
   ```bash
   python build-index.py
   ```

   This reads all `.md` files from `raw/`, chunks them intelligently, embeds them with `models/gemini-embedding-001`, and saves:
   - `embeddings/faiss_index.bin` - Vector similarity search index
   - `embeddings/chunks_metadata.json` - Chunk metadata and source tracking

**Dry-run mode** (preview without embedding):
```bash
python build-index.py --dry-run
```

### Environment Variables

- `GOOGLE_API_KEY` - Required for building the vector store (set in `.env` or environment)
- `KB_INDEX_PATH` - Path to FAISS index (default: `knowledge_base/embeddings/faiss_index.bin`)
- `KB_METADATA_PATH` - Path to metadata JSON (default: `knowledge_base/embeddings/chunks_metadata.json`)

## Usage in Backend

The backend (`nalam/backend.py`) automatically:

1. Loads the vector store at startup via `retriever.load_index()`
2. For each `/counselling` request, retrieves relevant knowledge chunks
3. Injects retrieved context into the prompt before LLM generation
4. Falls back gracefully to zero-shot generation if vector store is unavailable

Example:
```python
from knowledge_base import retriever

# At startup
if retriever.load_index():
    print("RAG enabled")
else:
    print("RAG disabled; using zero-shot generation")

# During /counselling
rag_query = retriever.build_rag_query(danger_signs, visit_day, weight, extra_concerns)
chunks = retriever.retrieve(rag_query, top_k=4)
context = retriever.format_context(chunks)
# → context injected into prompt
```

## Documentation Sources

All raw documents follow clinical HBNC guidelines:

1. **01-hbnc-visit-schedule-and-danger-signs.md** - Visit schedules, danger signs, monitoring checklist
2. **02-nutrition-and-diet-recommendations.md** - Breastfeeding, nutrition by visit day, Tamil Nadu guidelines
3. **03-infection-signs-and-home-care.md** - Sepsis, cord infection, home prevention
4. **04-followup-schedule.md** - Follow-up timing, referral pathways
5. **05-visit-report-template.md** - Visit report structure and required fields

## Vector Store Details

- **Model**: `models/gemini-embedding-001` (3072-dimensional vectors)
- **Total chunks**: ~386 chunks from 5 source documents
- **Chunking strategy**: Intelligent splitting by markdown headers (`##`) then by 500-char limit respecting paragraph boundaries
- **Index type**: FAISS IndexFlatL2 (L2 distance for similarity)
- **Metadata**: Chunk ID, source file, heading, text length for full traceability

## Quota and Rate Limits

The Gemini free tier provides 100 embedding requests/minute. The builder:
- Batches 50 chunks per request
- Spaces batches 20+ seconds apart
- Implements retry logic with exponential backoff for quota errors

For production use, upgrade to a paid API key for higher quotas.
